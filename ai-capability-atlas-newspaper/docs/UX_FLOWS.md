# UX ARCHITECTURE

## PRIMARY JOURNEY
1. "I need a permit" option
2. Simple decision tree for selection
3. Step-by-step form completion
4. Submission with tracking dashboard

## EDGE CASES 
1. User doesn't know if permit is needed
2. User selects wrong permit type
3. Network drops during form fill
4. User leaves mid-form and returns later
5. Required document missing
...

## ERROR RECOVERY
- Clear instructions for every failure
- Offline drafts and autosave

## VALIDATION RULES
- Required and optional fields marked
- Simple format requirements
- Error messages at a 4th-grade level

## ZERO-FRICTION PRINCIPLES
- Minimize taps and inputs
- Smart defaults
- Auto-save
- Pre-fill recurring information
...
