# PRD Template

## 1. Problem Statement
- Clearly define the problem to be solved.

## 2. Goals
- List the measurable outcomes desired.

## 3. Requirements
### Functional Requirements
- What the system must accomplish.

### Non-Functional Requirements
- Performance, security, scalability attributes.

## 4. Constraints
- Technical, business, timeline, resource limitations.

## 5. Use Cases
- Describe key user flows and scenarios.

## 6. Success Metrics
- KPIs and metrics to measure success.

## 7. Out of Scope
- Define what is not being built.
